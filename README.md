
# Royal Games API

\- A **Royal Games API** foi um proposta de projeto escolar, para desenvolver algumas habilidades com **SQL Server**, **.NET Core**, **JWT**,   e reforçar os conceitos de API **(Aplication Programming Interface)**, seguindo os conceitos de DDD **(Drive Domain Design)** para modelagem da mesma.

## Habilidades desenvolvidas
- Lógica de programação
- SQLServer + automação de processos
- API desenvolvida em camadas
- Arquitetura de modelagem DDD
- .NET Core
- JWT  

## Finalidade
\- O projeto **RoyalGamesAPI** é uma das partes principais do projeto com o mesmo nome, que de forma enxuta é um sistema que vai desde mostruário de produtos para venda, até a parte de controle de mercadoria por administradores.

## Conceitos
\- No desenvolvimento do projeto, nos deparamos com conceitos muito abrangentes, que vão além do CRUD **(CREATE, READ, UPDATE e DELETE)**, Conceitos como, **Levantamento de requisitos**, **Regras de negocios**, UML **(Unfied Modeling Language)**, e **DDL (Data, Definition, Language)**, para concluir e melhorar a modelagem do banco de dados, de modo que ficasse normalizados, e sem redundância de dados, autenticação de usuarios por token **JWT**, validação de horários e informações do usuario logado para edição de produtos e demais tabelas, metodologias ageis como: **Kanban**, conceitos de **SCRUM**  _ex_: sprints e daylis para melhorar a comunicação do grupo. Além de ser um projeto, que possui muitas features e funções que divergem em codigo, assim então, sendo necessário uma boa base de conhecimento de **GitFlow**, e versionamento de codigo.
